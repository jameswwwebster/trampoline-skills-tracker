# Competition Category Age Brackets Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add optional age ranges to competition categories, filter eligible gymnasts by calendar-year age server-side, and improve the invite UI with a name search bar and age bracket tabs.

**Architecture:** Add `minAge`/`maxAge` to the `CompetitionCategory` schema via a single migration; update the Joi schemas and Prisma calls in the category create/update endpoints; extend the eligible endpoint to filter by age and return age fields; add age inputs to both admin category UIs; add name search and age tabs to the InvitesTab.

**Tech Stack:** Express + Prisma 5 + PostgreSQL (backend), React 18 (frontend), Joi validation, `bookingApi.js` axios wrapper

```jsx
                : notInvited;
              return filtered.length === 0
                ? <p className="bk-muted" style={{ fontSize: '0.85rem' }}>No gymnasts match your search.</p>
                : (
                  <table className="bk-table">
                    <tbody>
                      {filtered.map(g => renderGymnastRow(g, null))}
                    </tbody>
                  </table>
                );
            })()}
```
